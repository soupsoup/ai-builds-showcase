# AnthonyWiki

A personal knowledge wiki compiled from journals, notes, and messages — the context layer that Chief of Staff and Networker both read from when they need to know who you are, what you care about, or who you know.

## The problem

AI tools are good at synthesis and bad at memory. Every new session starts from zero unless you re-explain your context, which means either re-typing the same background constantly or accepting shallow answers. AnthonyWiki exists to give Claude (and the tools built on top of it) durable, structured context it can read instead of asking for.

## How it's built

The wiki is generated by a Claude Code skill (`/wiki`) that treats the job as writing, not filing. It runs in four stages:

- **Ingest** — converts raw source data (Day One journals, Apple Notes, Obsidian vaults, Notion exports, iMessage logs, email, even Twitter archives) into one markdown file per entry, with YAML frontmatter for date, source type, and tags. This step is mechanical — a Python script, no model calls.
- **Absorb** — the core step. Entries are processed chronologically; for each one, the skill re-reads the master index and every article it touches before deciding whether to update an existing article or create a new one. The instruction that matters here: the question is never "where does this fact go," it's "what does this mean, and how does it connect to what's already known." Wiki structure (which directories and articles exist) emerges from the data — nothing is pre-templated.
- **Query / cleanup / breakdown** — once compiled, the wiki supports natural-language queries against itself, an audit pass to catch thin or stale articles, and a "breakdown" pass that finds concepts mentioned across multiple entries that deserve their own article but don't have one yet.

The output is a directory of markdown articles plus `_index.md` (a master index with aliases) and `_backlinks.json` (a reverse link index), so any article can answer "what else references this" without a full-text search.

## The reader app

The wiki itself is a Next.js app (App Router, Tailwind v4) that compiles markdown content (via `remark`/`rehype`, with GFM support) into a browsable site — wiki-style cross-linking between people, projects, and companies, the same pattern Wikipedia uses for its own internal links.

## What's real vs. synthetic in this demo

The ingestion pipeline, absorption logic, and rendering app are unchanged. The demo runs against entirely fictional content — a fake person, fake company, fake project — generated to exercise the same cross-linking and article structure the real wiki produces, without exposing actual journal entries, contacts, or message history.
